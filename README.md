## `01` &nbsp; About

I'm a **BCA student** (4th semester) who builds things that live at the intersection of mobile, web, desktop, and hardware. My curiosity doesn't stop at one layer of the stack — I'm equally at home writing Jetpack Compose layouts, wiring Django REST endpoints, sketching GLSL fragment shaders, and programming Arduino sensors.

Right now I'm focused on:

- 🟣 &nbsp; Native Android apps with **Kotlin + Jetpack Compose**
- 🔵 &nbsp; Full-stack web products with **React + Django**
- ⚪ &nbsp; Real-time 3D scenes with **WebGL / custom GPU shaders**
- 🟢 &nbsp; Embedded IoT systems via **Arduino**

> *I believe the best interfaces disappear — leaving only the experience.*

## `03` &nbsp; Featured Projects

<!-- 3D Immersive Portfolio ──────────────────────────────────────── -->
<details>
<summary><b>◈ &nbsp;3D Immersive Portfolio</b> &nbsp;—&nbsp; <i>WebGL · Three.js · Custom GLSL Shaders</i></summary>
<br/>

A fully immersive portfolio experience built from scratch in the browser. Custom vertex and fragment shaders power the environment; users navigate a 3D scene rather than scrolling a page. Every surface, lighting cue, and transition is authored directly in GLSL — no asset store shortcuts.

**Key technical decisions:** procedural geometry, post-processing bloom pipeline, and a custom camera rig for smooth first-person navigation.

[![Live Demo](https://img.shields.io/badge/Live_Demo-4C566A?style=flat-square&logo=vercel&logoColor=88C0D0)](https://portfolio-aasignment.vercel.app)

</details>

<br/>

<!-- Restaurant Management System ─────────────────────────────────── -->
<details>
<summary><b>◈ &nbsp;Restaurant Management System</b> &nbsp;—&nbsp; <i>Java · OOP · SQL</i></summary>
<br/>

A rigorous object-oriented desktop application managing the full operational lifecycle of a restaurant — orders, inventory, billing, and staff — through clean domain modelling. Designed around SOLID principles with a layered architecture separating concerns across entity, service, and presentation layers.

**Key technical decisions:** normalized relational schema, DAO pattern for persistence, and a terminal UI optimized for speed-of-use in a real kitchen environment.

</details>

<br/>

<!-- SkillBridge ─────────────────────────────────────────────────── -->
<details>
<summary><b>◈ &nbsp;SkillBridge</b> &nbsp;—&nbsp; <i>Business Platform Pitch · React · Django</i></summary>
<br/>

A platform pitch designed to bridge the gap between skill-seekers and domain experts — think mentorship marketplace meets micro-learning. Built end-to-end as a pitch prototype with a working React frontend, Django REST API, and PostgreSQL backend.

**Key technical decisions:** JWT auth flow, real-time notification hooks, and a matching algorithm prototype based on skill-tag proximity.

</details>
